Column edit template
For more information, see Records editing.
Built-in edit template
1. Simple mode as attribute edit-type for ax-column template 

        2. or with more details as ax-column-edit template
        

        
Attributes:
        bind-to the property name (field) of row object (dataItem - as scope variable), if is not the same as ax-column parent element. Required if you want to use a built-in type view template.
        ng-model if you need to bind a property which is not a direct property of dataItem (row object) you must use ng-model.
        edit-type can be one of ax-framework controls implementation: text, text-with-zoom, date, datetime, checkbox, dropdown-list
                or browser native input types implementation: input-text, input-number, input-checkbox, input-radio, input-range, input-date, input-datetime or custom for particulary implementation.
        ng-disabled expression to conditional disable control input.
        tooltip with no value if you want to use getTooltipFor method to retrieve tooltip html content, or expression which evaluated for an uib-popover directive.
        style the css style for ax-column-edit template. Here you can use padding-left:0 to eliminate the default padding-left, or overflow-y=hidden if you have same problem with display elements, and the default overflow-y=scroll is not suitable for the case
        
ng-focus, ng-blur, ng-change attributes cannot be used for built-in ax-column-edit templates, will be overwrite. If you needed, you must use custom type for ax-column-edit template.
        

Custom edit template:
        
        

            

            For custom type, the current row object is named dataItem, which can be used to create your custom edit. You can access the
            ax-table parent scope with $ctrl.$parent ($ctrl.$parent.myVariable).
            
Custom edit can be used for edit nested properties, custom actions or anything else. This accept any html you need.
            
has-input - it's necessary for tab navigation between controls: only controls with has-input attribute will receive focus
            
ng-focus = "$ctrl.objectHasFocus($event, $parent.$parent.dataItem, fieldName);" - it's necessary for tab navigation too. This method add hasFocus class for current focused control, and remove  for previous. Also set currentColumnIndex, and currentField ax-table properties.
            

            ng-blur="$ctrl.$validateField(fieldName, $parent.$parent.dataItem)" - it's recommended to use $validateField method for validate Field data on blur.
