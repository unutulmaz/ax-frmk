Column template
axTable columns are defined with ax-column template like this: 

                Attributes:
                bind-to the property name (field) of row object (dataItem - as scope variable). Required if you need to sort ,filter or grouping with this column.
                invariant-field the invariant field for bindTo field of row object (dataItem - as scope variable). It is used fort sorting, filtering, or grouping. You can use the adapter (see js tab), to create invariant column.
                sortable if no value given, is default bind-to field, else will be an invariant column.
                header required, column title - used to display in sort, group, toggle hide columns popups, this must be an unique value (is used as column identifier).
                header-title column label (optional). Column can have multiple headers. Each header can have his own header-title (in this case is used only if ax-column-header template is not defined).
                show-header = 'false' when you don't wanna display column header (and no ax-column-header template it's defined).
                header-menu = 'false' when you don't wanna create column menu.
                width required. You can have just one column with 100%, the rest must be in pixels
                column-type can be one of values string, number, date,datetime,boolean-checkbox,boolean-radio,gutter-icons,crud-buttons. If not view-type attribute provided, or ax-column-view template, view-type will take value from column-type.
                view-type can be one of values as column-type attribute values. This will create a ax-column-view template with type = view-type. It's now need to use this if you set a column-type attribute.
                double-binding can be true for built-in view template, if column could be changed by user editing in other column. In usual case, one time binding is used for column view template.
                edit-typecan be one of values text, text-with-zoom, date, datetime, checkbox, dropdown-list
                or browser native input types implementation: input-text, input-number, input-checkbox, input-radio, input-range, input-date, input-datetime.
                This will create a ax-column-edit template with type = edit-type. It's now need to use this attribute if column is not editable.
                edit-tooltip if column is editable, a tooltip to help user to understand what is about in the column.
                show-in-editor = false for hide/show column in editor. This attribute overwrite hidden column state, which also hide/show column in editor if show-in-editor attribute is not defined.
                editor-column-index column-index (1 based) where to add control in editor, when editor has more than one column for arrange controls (columns-no attribute of ax-grid-editor).
                filter-menu the built-in filter options for data type: string, number, date, datetime, boolean
                initial-filter-option the initial selected filter type for the column.
                hideable = "false" When you need to interdict column hidden, otherwise column is hideable
                hidden-column if you need to hide column, otherwise si visible.
                exportable = "false" if you need to disabled column for export, otherwise column is exportable.
                class the css class for ax-column-view template - if not is explicitly defined.
                style the css style for ax-column-view template - if not is explicitly defined.
                date-format date-format for ax-column-view template - if not is explicitly defined
                datetime-format datetime-format if view-type=datetime for ax-column-view template - if not is explicitly defined
                locale locale format for view-type=number if view-type=number for ax-column-view template - if not is explicitly defined
                decimals decimals number for view-type=number if view-type=number for ax-column-view template - if not is explicitly defined
                

ax-column element has some templates for provide special header, view, edit or filter capabilities:
                
- ax-column-header
                
- ax-column-view
                
- ax-column-edit
                
- ax-column-editor
                
- ax-column-filter
                
- ax-column-filter-menu
