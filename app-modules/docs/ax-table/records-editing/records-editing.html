Records editing
For edit data inside ax-table, you need to
1. Set axTable attributes: edit-row with one of these values inline, inline-cell or popup. apply-changes-on-save true or false. By default is false for data-table and true for data-list. For edit-row = inline-cell is false and not assignable.
2. Add some button to ax-toolbar button for add new records button to enable edit mode for edit-row=inline-cell type.
3. Add two columns: for global messages error icon and save icon (which is shown if data is saved, and disapear after 300ms). If you don have global record error message and don't want semnalize to user when data si saved, you don't need to add this column. a column with buttons: view, edit, delete, save and undo, depending on edit-row type, and operation executed by the user.
4. Setup ax-column-edit template for each column you need for data edit.

Inline edit mode
- In this mode, editing controls are shown after user press edit button from Actions column, or pressing F2.
- Data is saved when user press save button from Actions column, or when press Ctrl+S. Use can also choose to undo changes (Escape). Calculations are updated after save.
- User can edit just one row at the time.
- If apply-changes-on-save = false user must press Apply changes Settings button for order and filter apply.

Inline-cell edit mode
- In this mode, editing controls are shown after user press Edit Data button from toolbar.
- Data is saved when user leave current control. User cannot undo changes. Calculations are updated after save.
- Data is not saved if validations return errors. Row is marked as having errors - in gutter icons column and in the column with error are shown errors icons.
- User can edit current row.
- User must press Apply changes button for order and fitler apply.
