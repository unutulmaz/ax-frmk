Records filtering Column filter Column filter menu
Records filtering can be made by end user with Global search input box control, or by choosing from Column filter menu a control for filter. A filter menu (ax-column-filter-menu) is defined with ax-column-filter elements as options.

Global search is defined with an ax-global-search toolbar element (can be used only inside ax-table ax-toolbar). 

		You can specify which columns are searched by default, by search-columns attribute: a list separated by comma. User can change this by selecting columns he wants, in Global search options popup.
		
If you want to create your own, you need to store value in $ctrl.filters.globalSearch, and for filter apply use method $ctrl.filterApply() without arguments.
		Columns list must be stored in $ctrl.filters.config.globalSearch.columns. You can also use method $ctrl.itemCustomFilter for custom filtering.
