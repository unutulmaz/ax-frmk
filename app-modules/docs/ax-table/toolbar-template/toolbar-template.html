Toolbar template
Toolbar is defined with an ax-toolbar template. axTable element accept one ax-toolbar template as child element. In this template you can use some built-in controls, for add axTable commands.
AxToolbar is a scrollable element: has an ax-scroller directive which permit to split toolbar in three sections: left (toolbar = left), right (toolbar = right), and in the middle (no toolbar attribute) a scrollable content.
With min-width toolbar attribute you can fix when toolbar is shown in a popup for mobile devices.
You can disable adding ax-scroller directive including ax-scroller = false attribute for ax-toolbar. 

				ax-button Button-type values
				
- refresh  - button for load data (when you define an api-controller for ax-table)
				
- check-all  - button for select all records when selectable-rows = multiple
				
- uncheck-all  - button for clear selection records when selectable-rows = multiple
				
- initial-select  - button for execute method loadInitialData view controller method for reports
				
- groups-toggle  - button for toggle collapse/expand records grouping
				
- maximize  - button for toggle maximize/restore ax-table on his parent
				
- settings  - button for display ax-table setting options. This button has an attribute show-title which can control if button title is shown or not. Below are some ax-table attributes which enable/disable settings buttons
				
- customizable-config true/false for show/hide config option.
				
- customizable-pivot-table true/false for show/hide pivot table option.
				
- columns-autofit-enabled true/false for show/hide all columns autofit widths option.
				
- export-disabled true/false for show/hide export options.
				
- customizable-edit-mode true/false for show/hide edit mode options.
				
- show-change-edit true/false for show/hide change edit button for inline-cell edit mode.
				
- can-add true/false for show/hide add button button.
				
- apply-changes-on-save true/false for show/hide apply changes button.

				

ax-global-search - object for create a input box for global search

				
Developer can overwrite ng-click event and button innerHTML (label).
				

To use these built-in toolbar object is optional.
