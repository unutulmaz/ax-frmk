Position and size in page
axTable element position is given by style position attribute
- style = "position:relative;width:50%;height:100%" width and height can be set as percent or as pixels.
- style = "position:absolute;top:40px;left:10px;right:10px;bottom:10px". In this case, a common mistake is using in same time width=100%, or height=100%.
- style = "position:absolute;top:0px;left:0px;right:0px;bottom:0px" is default style for ax-table element
The maximize button change axTable style to "position:absolute;top:0;bottom:0;left:0;right:0;z-index:100;margin:0" and trigger a layout method to refresh axTable dimensions. This will not work in any structure, so, you can create your own method.
If you resize programmatically a parent element, which affected axTable dimensions, you must trigger a window resize event for all axTables rearranged in new workspace like this:
$timeout(axUtils.triggerWindowResize) where axUtils.triggerWindowResize = function(){window.dispatchEvent(new Event('resize'));}
